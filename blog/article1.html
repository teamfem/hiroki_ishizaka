FEniCSx を Google Colab で動かし，相対誤差と可視化を確認する

2026年4月12日

このページでは，FEniCSx で書かれた研究用コードを出発点として，主問題の P1 有限要素近似だけを切り出し，Google Colab 上で動かせる形に整理した流れをまとめます． あわせて，相対誤差 \[ \frac{|u-u_h|_{H^1}}{|u|_{H^1}},\qquad \frac{\|u-u_h\|_{L^2}}{\|u\|_{L^2}} \] の確認，メッシュ分割図の表示，近似解 \(u_h\) の 2D 可視化，さらに 3D surface plot まで行います．

1．出発点となったコード

最初に確認した元コードは，単なる PDE ソルバではなく，FEniCSx 上で半線形楕円問題 \[ -\Delta u + u^3 = f \quad \text{in }\Omega=(0,1)^2, \qquad u=0 \quad \text{on }\partial\Omega \] を解きつつ，検証半径や目的量の認証まで行う研究用スクリプトでした． その中から，まずは主問題を解く部分だけを抜き出して，標準的な誤差収束を確認できるように整えました．

2．主問題だけを取り出した簡潔版

主問題では，空間離散として P1 の Lagrange 有限要素を用い，非線形項 \(u^3\) は Newton 法で処理します． 弱形式は \[ \int_{\Omega} \nabla u_h\cdot\nabla v_h\,dx + \int_{\Omega} u_h^3 v_h\,dx = \int_{\Omega} f v_h\,dx \qquad (\forall v_h\in V_h) \] です． 今回の確認では，厳密解として \[ u(x,y)=\sin(\pi x)\sin(\pi y) \] を用い，それに対応する右辺 \[ f=2\pi^2 u + u^3 \] を作りました． この設定なら，有限要素近似 \(u_h\) の誤差を直接測ることができます．

3．相対誤差の定義

今回は誤差を絶対誤差ではなく，相対誤差で見ました．具体的には

\[ E_{H^1}^{\mathrm{rel}} = \frac{|u-u_h|_{H^1}}{|u|_{H^1}}, \qquad E_{L^2}^{\mathrm{rel}} = \frac{\|u-u_h\|_{L^2}}{\|u\|_{L^2}} \]

です．厳密解が十分滑らかで，P1 有限要素を用いているので，観測される収束率としては通常

  • relative H1 seminorm error \(\sim O(h)\)
  • relative L2 norm error \(\sim O(h^2)\)

が期待されます．したがって，メッシュを細かくしたときに，相対 \(H^1\) 半ノルム誤差の rate が 1 に近づき，相対 \(L^2\) 誤差の rate が 2 に近づけば，実装が筋よく動いていると判断しやすくなります．

4．可視化の追加

相対誤差だけでなく，結果が目で確認できるように次の可視化も追加しました．

  • 粗いレベルでのメッシュ分割図
  • 最も細かいレベルでの近似解 \(u_h\) の 2D プロット
  • 最も細かいレベルでの近似解 \(u_h\) の 3D surface plot

2D 領域上のスカラー値 \(u_h(x,y)\) を 3D で描くときは，\((x,y)\) を平面座標，高さを \(u_h\) にした surface plot にします．これは 3 次元問題を解いているという意味ではなく，2 次元領域上の解の形を見やすくするための表示です．

5．Google Colab での実行手順

ここからは，Google Colab 上で実際に動かす手順をまとめます． まず，ChatGPT で作成した Python ファイルをローカルに保存しておきます．今回は最終的に extract_fem_primal_with_relative_error_checks_and_plots_colab_fix4_3d.py を使いました．

5.1 FEniCSx のインストール

Colab では最初に FEniCSx を導入します．最初のセルで次を実行します． 

try:
    import dolfinx
except ImportError:
    !wget "https://fem-on-colab.github.io/releases/fenicsx-install-release-real.sh" -O "/tmp/fenicsx-install.sh"
    !bash "/tmp/fenicsx-install.sh"
    import dolfinx

print("FEniCSx ready")

5.2 導入確認

import dolfinx
import ufl
from mpi4py import MPI
from petsc4py import PETSc

print("dolfinx OK")

5.3 スクリプトのアップロード

左側のファイル欄からアップロードしてもよいですし，セルから次を実行してアップロードしてもかまいません． 

from google.colab import files
uploaded = files.upload()

5.4 実行

!python extract_fem_primal_with_relative_error_checks_and_plots_colab_fix4_3d.py

実行後には，相対誤差表と画像ファイルが出力されます．

5.5 出力確認

!ls -l

画像をノートブック上に表示するには，例えば次を使います． 

from IPython.display import Image, display

display(Image(filename="uh_level_4.png"))
display(Image(filename="uh_level_4_3d.png"))

6．途中で出たエラーと修正点

今回は，計算本体よりも可視化まわりでいくつか修正が必要でした．

6.1 出力先 /mnt/data の問題

Colab では /mnt/data が最初から存在しない場合があるため， 

FileNotFoundError: [Errno 2] No such file or directory: '/mnt/data/fem_relative_error_rows.tex'

というエラーが出ました．これは FEniCSx の問題ではなく，単に出力先ディレクトリの問題です．最終版では，出力先をカレントディレクトリにするか，自動作成するようにしました．

6.2 dof 座標の reshape 問題

次に，可視化用の自由度座標を reshape(-1, 2) のように固定して扱っていたため，Colab の環境で 3 成分座標が返るケースに当たり，描画部分が落ちました．計算自体は終わっていたので，問題は plot 関数だけでした．

6.3 V.dofmap.list.array の API 差

さらに，FEniCSx のバージョン差により， V.dofmap.list.array がある環境と， V.dofmap.list そのものが NumPy 配列で返る環境がありました． この差を避けるために，最終版では内部表現に直接触れず， dolfinx.plot.vtk_mesh(V) を使って描画用の三角形分割を作るようにしました．これで Colab 上でも安定しやすくなりました．

7．出力されるファイル

最終版を実行すると，例えば次のようなファイルが得られます．

  • fem_relative_error_rows.tex：相対誤差と実験収束率の表
  • mesh_level_0.png：粗いメッシュ分割図
  • uh_level_4.png：近似解の 2D 可視化
  • uh_level_4_3d.png：近似解の 3D surface plot

これらはそのままノートやスライド，あるいは研究メモに流用しやすい形です．

8．今回の流れのまとめ

  1. 研究用の FEniCSx コードを確認し，主問題の有限要素部分を取り出した．
  2. 絶対誤差ではなく，相対 \(H^1\) 半ノルム誤差と相対 \(L^2\) 誤差を出す形にした．
  3. メッシュ図と近似解の 2D 可視化を加えた．
  4. さらに 3D surface plot を加えた．
  5. Google Colab で動かすために，導入手順と実行手順を整理した．
  6. Colab 固有のファイル出力や可視化まわりのエラーを修正した．

今回の作業は，「複雑な研究コード全体をいきなり動かす」のではなく，まず主問題・誤差確認・可視化という最小限で堅い部分を切り出して整える，という意味でも有益でした． 研究用コードを整理するときは，このように小さな確認可能単位へ分解するのが実用的です．
