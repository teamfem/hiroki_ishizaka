Google ColabでFreeFEMを使う：インストールからPoisson方程式の可視化まで

Introduction

Google Colab は，ブラウザから気軽に計算実験を始められる便利な環境です．一方で，有限要素法の実験では，どのソフトウェアをどう入れるかで最初に少し迷いやすいです．このページでは，Google Colab 上で FreeFEM を使い，最小の動作確認から Poisson 方程式の計算，さらに Python による可視化まで行う手順をまとめます．

今回の目標は 4 つです．

1．Google Colab 上で FreeFEM を動かすための基本手順を確認すること．
2．FreeFem++-nw コマンドで .edp ファイルを実行する流れに慣れること．
3．Poisson 方程式の最小例を通じて，FreeFEM で有限要素法を解く基本の形を見ること．
4．Colab では GUI 表示よりも，数値を書き出して Python 側で図示する流れが実用的であることを確認すること．

ここで扱う問題は単位正方形 \[ \Omega=(0,1)^2 \] 上の Poisson 方程式 \[ -\Delta u = 1 \quad \text{in } \Omega, \qquad u=0 \quad \text{on } \partial\Omega \] です．厳密解を使って誤差評価まで進む段階の手前として，まずは「Colab で FreeFEM が動き，有限要素解が計算でき，図として確認できる」ことを目標にします．

Google Colabでの実行手順

まず Google Colab を開き，新しいノートブックを作成します．この例では GPU は不要なので，通常の CPU ランタイムで十分です．有限要素法の計算自体は FreeFEM に任せ，図の作成は Python に任せるので，Colab の標準的な構成のままで進められます．

Colab はセッションごとに環境が初期化されることがあるため，FreeFEM を毎回インストールする前提でセルを用意しておくのが分かりやすいです．また，環境構築と動作確認を分けておくと，エラーが出たときの切り分けもしやすくなります．その意味で，以下の 4 セル構成にしておくと再利用しやすいと思います．

セル1：FreeFEM のインストール

最初のセルでは，Ubuntu ランタイムのバージョンを確認し，対応する FreeFEM の .deb パッケージを入れます．このセルは環境構築だけに徹し，動作確認は次のセルに分離します．こうしておくと，どこで失敗したかが分かりやすくなります． 

%%bash
set -e

apt-get update -y
apt-get install -y wget ca-certificates \
  libgsl-dev libhdf5-dev liblapack-dev libopenmpi-dev freeglut3-dev

source /etc/os-release
echo "Ubuntu version: $VERSION_ID"

if [ "$VERSION_ID" = "24.04" ]; then
  URL="https://github.com/FreeFem/FreeFem-sources/releases/download/v4.15/FreeFEM-4.15-amd64-ubuntu24.04.deb"
elif [ "$VERSION_ID" = "22.04" ]; then
  URL="https://github.com/FreeFem/FreeFem-sources/releases/download/v4.15/FreeFEM-4.15-amd64-ubuntu22.04.deb"
else
  echo "Unsupported Ubuntu version in this Colab runtime: $VERSION_ID"
  exit 1
fi

wget -q "$URL" -O /tmp/freefem.deb
dpkg -i /tmp/freefem.deb || apt-get -f install -y

echo "Installed binaries:"
which FreeFem++ || true
which FreeFem++-nw || true
ls -l /usr/local/bin/FreeFem* || true

このセルでは，まだ .edp ファイルは実行しません．インストール確認セルの最後で存在しないスクリプトを呼んでしまうと，原因の切り分けがしにくくなるからです．Colab では，環境構築と疎通確認を分けておく方が安全です．

セル2：最小の動作確認

次に，ごく小さい .edp ファイルを書いて，FreeFEM が本当に動くかを確かめます．ここでは正方形メッシュを作り，頂点数と三角形数を表示するだけです．ファイルは明示的に /tmp/hello.edp に書き，その同じパスを実行します． 

%%bash
cat > /tmp/hello.edp <<'EOF'
cout << "Hello from FreeFEM" << endl;
mesh Th = square(4,4);
cout << "number of vertices = " << Th.nv << endl;
cout << "number of triangles = " << Th.nt << endl;
EOF

FreeFem++-nw /tmp/hello.edp

このセルの役割は単純です．FreeFEM の実行系が壊れていないことと，.edp をセル内から生成して実行できることを確かめています．もしここで止まるなら，Poisson 問題に進む前に，ファイルの置き場所やコマンド名を見直せばよいです．

セル3：Poisson 方程式を解いて格子点データを書き出す

ここで，Poisson 方程式 \[ -\Delta u = 1, \qquad u|_{\partial\Omega}=0 \] を単位正方形上で解きます．FreeFEM の solve 構文を使い，その後，格子点上の値をテキストファイルに書き出します．Colab では FreeFEM の対話的 GUI 描画をそのまま使うよりも，こうして数値を外へ出し，Python で描き直す方が安定です． 

%%bash
cat > poisson.edp <<'EOF'
verbosity = 2;

mesh Th = square(40, 40);

fespace Vh(Th, P1);
Vh u, v;

func f = 1.0;

solve Poisson(u, v)
    = int2d(Th)( dx(u)*dx(v) + dy(u)*dy(v) )
    - int2d(Th)( f*v )
    + on(1, 2, 3, 4, u=0);

ofstream out("u_samples.txt");
int N = 101;

for (int j = 0; j < N; j++) {
    real y = j * 1.0 / (N - 1);
    for (int i = 0; i < N; i++) {
        real x = i * 1.0 / (N - 1);
        out << x << " " << y << " " << u(x,y) << endl;
    }
}

cout << "u(0.5,0.5) = " << u(0.5,0.5) << endl;
cout << "Done." << endl;
EOF

FreeFem++-nw poisson.edp

このコードでは，mesh Th = square(40,40); で単位正方形の三角形分割を作り，fespace Vh(Th, P1); で 1 次有限要素空間を定義しています．その上で，弱形式 \[ \int_\Omega \nabla u\cdot\nabla v\,dx = \int_\Omega fv\,dx \] を solve でそのまま書いています．

最後に u_samples.txt を出力しているのは，Colab 上で図示しやすくするためです．要するに，有限要素法で解くところまでは FreeFEM，見やすい図を作るところは Python，と役割分担しています．

セル4：Python で可視化

セル 3 で書き出した u_samples.txt を NumPy で読み込み，Matplotlib で等高線図を描きます． 

import numpy as np
import matplotlib.pyplot as plt

data = np.loadtxt("u_samples.txt")
x = data[:, 0]
y = data[:, 1]
u = data[:, 2]

N = int(np.sqrt(len(u)))
X = x.reshape(N, N)
Y = y.reshape(N, N)
U = u.reshape(N, N)

plt.figure(figsize=(6, 5))
cp = plt.contourf(X, Y, U, levels=30)
plt.colorbar(cp)
plt.xlabel("x")
plt.ylabel("y")
plt.title("FreeFEM solution of -Δu = 1 on the unit square")
plt.tight_layout()
plt.show()

これで，境界で 0 になり，内部で山型になる Poisson 解の典型的な形が確認できます．最初は「解けたかどうか」だけでも十分ですが，図が出ると計算対象の感覚が一気につかみやすくなります．

各セルの役割をどう理解するか

この 4 セルは，それぞれ別の役割を持っています．

セル 1 は環境構築です．Colab 上で FreeFEM を使える状態にします．
セル 2 は疎通確認です．最小の .edp を通して「FreeFEM が生きている」ことを見ます．
セル 3 は有限要素法の本体です．弱形式を書いて解き，さらにデータを書き出します．
セル 4 は可視化です．Colab で見やすい図にするために Python を使います．

この分け方にしておくと，あとからコードを拡張しやすいです．たとえば，右辺を変える，境界条件を変える，メッシュ数を変える，誤差評価を加える，などの変更は主にセル 3 に集中します．一方で，セル 1 とセル 2 はほぼそのまま流用できます．

FreeFEM の記法の最小限の読み方

FreeFEM のコードは，一度慣れると非常に直接的です．今回の例で大事なのは次の点です．

mesh Th = square(40,40); は，単位正方形を分割したメッシュを作ります．
fespace Vh(Th, P1); は，そのメッシュ上の P1 有限要素空間を作ります．
Vh u, v; で，数値解と試験関数に相当する有限要素関数を用意します．
solve Poisson(u, v) = ... の部分で，弱形式をほぼ数式どおりに書きます．
on(1,2,3,4,u=0) は，境界ラベル 1,2,3,4 上で同次 Dirichlet 条件を課しています．

ここで「数式に近い形で書ける」というのが FreeFEM の大きな利点です．とくに，有限要素法の講義や実験ノートでは，理論とコードの対応が見やすいです．

なぜ Colab では Python 可視化が便利なのか

FreeFEM には描画機能がありますが，Colab のようなブラウザ実行環境では，ローカルデスクトップのような感覚でそのまま使えるとは限りません．そのため，数値解を一度テキストやメッシュファイルへ書き出し，Python 側で読み込んで図示する方が扱いやすいです．

この方法の利点は 3 つあります．

1．Colab 上で安定して図を表示しやすいこと．
2．NumPy や Matplotlib にそのままつなげられること．
3．あとで誤差表や log-log プロットなども同じ Python ノートブックの中で扱えること．

将来的に，厳密解を入れて相対誤差を測る，複数のメッシュで収束率を出す，3 次元曲面図を描く，という拡張をする場合にも，この流れはそのまま使えます．

次に拡張するなら何をやるか

今回の記事は，まずは Colab で FreeFEM を動かすことに集中した最小構成です．この次の段階としては，次のような拡張が自然です．

1．右辺を厳密解に対応する形へ変えて，真の誤差を測ること．
2．メッシュを細かくしながら，\(L^2\) 誤差や \(H^1\) 型誤差の収束率を確かめること．
3．解の可視化を等高線図だけでなく 3 次元曲面図へ広げること．
4．境界条件や領域形状を変えて，より FreeFEM らしい記述に慣れること．

つまり，今回のノートブックは終点ではなく入口です．しかし，入口としてはかなり良いです．環境構築，有限要素法の計算，ファイル出力，可視化という一連の流れが短いコードで見えるからです．

このページの使い方

このページは，FreeFEM を Colab で試してみたい人のための最初のメモとして使えます．最初はそのままコピペで動かし，次にメッシュ数や右辺関数を書き換え，そのあと誤差評価を足していく，という使い方が自然です．

FEniCSx のときと同じく，「環境構築」「有限要素法の本体」「可視化」を分けておくと，どこで何をしているかが見えやすくなります．FreeFEM は数式とコードの距離が近いので，理論と実装を往復しながら学びたいときにとても相性がよいと思います．
