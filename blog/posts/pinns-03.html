第3回：どの幾何的量が誤差を支配するか

連載：PINNsはなぜ失敗するのか — 1次元から探る幾何条件の役割　|　27th June 2026

この回の目的

第2回では，残差点の配置を変えると PINNs の近似誤差が大きく変わること， そして最大ギャップ \(h_{\max}\) が誤差と相関するらしいことを数値的に観察しました．

しかし，これは5種類の配置を比べた段階の印象です． 次の問いに答えるには，より系統的な実験が必要です：

残差点の配置から計算できる幾何的量のうち，どれが PINNs の誤差を最もよく予測するか？

この回では，複数の配置族・複数の点数 \(N\) にわたって誤差を計測し， 5つの幾何的量との相関を系統的に調べます．

数値実験の前に，なぜ幾何的量が誤差に現れるかを，損失関数の構造から確認します．

なぜ幾何的量が現れるか：損失関数を積分として読む

PINNs の PDE 損失は $$ \mathcal{L}_{\rm pde}(\theta) = \frac{1}{N} \sum_{i=1}^{N} r_\theta(x_i)^2, \qquad r_\theta(x) := -u_\theta''(x) - f(x) $$ です．これは積分 $$ I[r^2] := \int_0^1 r_\theta(x)^2\, dx $$ の数値求積近似とみなせます．

数値求積の誤差は，点集合 \(\{x_i\}\) の均一性に依存します． これを定量化する理論が Koksma–Hlawka の不等式 です： $$ \left| \frac{1}{N} \sum_{i=1}^{N} g(x_i) - \int_0^1 g(x)\, dx \right| \leq D_N^* \cdot V(g). $$ ここで \(D_N^*\) は点集合のスター乖離度（star discrepancy）， \(V(g)\) は \(g\) の全変動です．

したがって，損失関数 \(\mathcal{L}_{\rm pde}\) が小さくなっても， それは \(I[r^2]\) が小さいことを意味するわけではありません． 乖離度 \(D_N^*\) が大きいほど，損失と真の積分の差が大きくなり得ます．

これが，残差点の配置が誤差に影響する根本的な理由です．

最大ギャップと乖離度の関係

スター乖離度と最大ギャップ \(h_{\max}\) の間には，次の関係があります： $$ \frac{h_{\max}}{2} \leq D_N^* \leq h_{\max}. $$ つまり，\(h_{\max}\) は \(D_N^*\) の上下界を与えます． 最大ギャップが大きければ乖離度も大きく，乖離度が小さければ最大ギャップも小さい．

有限要素法の誤差評価で最大要素径 \(h = \max_K h_K\) が主役を演じるのと 同じ構造が，ここにも現れています．

以下の数値実験で，これが実際の PINNs 誤差に対しても成立するかを確認します．

調べる幾何的量

残差点 \(0 < x_1 < \cdots < x_N < 1\) に対して， \(x_0 := 0,\, x_{N+1} := 1\) とおき， ギャップ列 \(\Delta_i := x_i - x_{i-1}\)（\(i=1,\ldots,N+1\)）を定義します．

今回調べる5つの幾何的量：

  • \(h_{\max} = \max_i \Delta_i\)：最大ギャップ． FEM の最大要素径に対応．
  • \(\bar{h} = 1/(N+1)\)：平均ギャップ． 点数のみで決まる基準値（一様分布なら \(h_{\max} = \bar{h}\)）．
  • \(\sigma_h = \mathrm{std}(\Delta_i)\)：ギャップの標準偏差． 配置の不均一さを測る．
  • \(\rho = h_{\max} / \min_i \Delta_i\)：最大最小比． FEM の形状正則性条件（異方性比）に対応．
  • \(D_N^*\)：スター乖離度． Koksma–Hlawka 不等式に直接現れる量．

セル 1：インポートと共通定義

import torch
import torch.nn as nn
import numpy as np
import matplotlib.pyplot as plt
from scipy import stats

torch.manual_seed(0)
np.random.seed(0)
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
print(f"device: {device}")

# 問題設定
def u_exact(x):
    return torch.sin(np.pi * x)

def f_rhs(x):
    return (np.pi ** 2) * torch.sin(np.pi * x)

# ニューラルネットワーク
class MLP(nn.Module):
    def __init__(self, hidden=32, n_layers=3):
        super().__init__()
        layers = [nn.Linear(1, hidden), nn.Tanh()]
        for _ in range(n_layers - 1):
            layers += [nn.Linear(hidden, hidden), nn.Tanh()]
        layers += [nn.Linear(hidden, 1)]
        self.net = nn.Sequential(*layers)
    def forward(self, x):
        return self.net(x)

# PDE 残差
def pde_residual(model, x):
    x = x.clone().requires_grad_(True)
    u = model(x)
    u_x  = torch.autograd.grad(u,   x, torch.ones_like(u),   create_graph=True)[0]
    u_xx = torch.autograd.grad(u_x, x, torch.ones_like(u_x), create_graph=True)[0]
    return -u_xx - f_rhs(x)

# 評価グリッド
x_test = torch.linspace(0, 1, 500).unsqueeze(1).to(device)
u_ref  = u_exact(x_test.cpu()).numpy().flatten()

セル 2：幾何的量の計算

def geometric_quantities(pts):
    """
    pts: numpy 配列，昇順，境界 0,1 を含まない内点
    戻り値: dict
    """
    pts_full = np.concatenate([[0.0], np.sort(pts), [1.0]])
    gaps = np.diff(pts_full)
    N = len(pts)

    h_max  = gaps.max()
    h_mean = 1.0 / (N + 1)          # 平均ギャップ（点数から決まる）
    h_std  = gaps.std()
    h_ratio = gaps.max() / (gaps.min() + 1e-12)

    # スター乖離度 D*_N（境界を含まない内点のみで計算）
    pts_s = np.sort(pts)
    i_arr = np.arange(1, N + 1)
    disc  = max(
        np.max(i_arr / N - pts_s),
        np.max(pts_s - (i_arr - 1) / N)
    )

    return {
        "h_max"  : h_max,
        "h_mean" : h_mean,
        "h_std"  : h_std,
        "h_ratio": h_ratio,
        "disc"   : disc,
    }

セル 3：配置生成関数（第2回を再利用・拡張）

def make_uniform(N):
    return np.linspace(0, 1, N + 2)[1:-1].astype(np.float32)

def make_random(N, seed):
    rng = np.random.default_rng(seed)
    return np.sort(rng.uniform(0, 1, N)).astype(np.float32)

def make_power(N, power):
    """t^power：power > 1 で左端集中，power < 1 で右端集中"""
    t = np.linspace(0, 1, N + 2)[1:-1]
    return (t ** power).astype(np.float32)

def make_chebyshev(N):
    i = np.arange(1, N + 1)
    return (0.5 * (1 - np.cos(np.pi * i / (N + 1)))).astype(np.float32)

def make_center(N, width):
    """全点を [(1-width)/2, (1+width)/2] に押し込む"""
    t = np.linspace(0, 1, N + 2)[1:-1]
    return ((1 - width) / 2 + t * width).astype(np.float32)

セル 4：訓練関数

def train_and_eval(pts, n_epochs=3000, lr=1e-3, lam=100.0, nn_seed=0):
    """
    残差点 pts で PINNs を訓練し，L2 誤差を返す．
    n_epochs=3000 は全体の実行時間を抑えるための設定．
    """
    torch.manual_seed(nn_seed)
    model = MLP(hidden=32, n_layers=3).to(device)
    opt   = torch.optim.Adam(model.parameters(), lr=lr)

    xc  = torch.tensor(pts, dtype=torch.float32).unsqueeze(1).to(device)
    xbc = torch.tensor([[0.0], [1.0]], device=device)
    ubc = torch.tensor([[0.0], [0.0]], device=device)

    for _ in range(n_epochs):
        opt.zero_grad()
        res  = pde_residual(model, xc)
        loss = (res**2).mean() + lam * ((model(xbc) - ubc)**2).mean()
        loss.backward()
        opt.step()

    with torch.no_grad():
        u_pred = model(x_test).cpu().numpy().flatten()
    return float(np.sqrt(np.mean((u_pred - u_ref)**2)))

セル 5：データ収集（10〜15分程度かかります）

多様な配置族・複数の \(N\) を組み合わせて約 50 件のデータを収集します． 

records = []   # {"label", "pts", geometry..., "err_L2"}

def record(label, pts):
    geom = geometric_quantities(pts)
    err  = train_and_eval(pts, n_epochs=3000)
    row  = {"label": label, **geom, "err_L2": err}
    records.append(row)
    print(f"{label:35s}  h_max={geom['h_max']:.4f}  "
          f"disc={geom['disc']:.4f}  L2={err:.3e}")

# 1) Uniform，N を変化させる
for N in [5, 8, 10, 15, 20, 30, 50]:
    record(f"uniform  N={N:3d}", make_uniform(N))

# 2) Random，N=30，シードを変化させる
for seed in range(10):
    record(f"random   N=30 seed={seed}", make_random(30, seed))

# 3) Left-clustered (べき乗)，N=30
for p in [1.5, 2.0, 3.0, 4.0, 6.0]:
    record(f"power={p:.1f}  N=30", make_power(30, p))

# 4) Chebyshev，N を変化させる
for N in [10, 15, 20, 30, 50]:
    record(f"chebyshev  N={N:3d}", make_chebyshev(N))

# 5) Center-clustered，幅を変化させる（N=30）
for w in [0.10, 0.20, 0.30, 0.50, 0.70, 0.90]:
    record(f"center  width={w:.2f}  N=30", make_center(30, w))

print(f"\n合計 {len(records)} 件のデータを収集しました")

結果の読み方

相関の強さで何がわかるか

Spearman 順位相関 \(\rho\) は，−1 から +1 の範囲を取り， \(\rho\) が 1 に近いほど「その量が大きいほど誤差も大きい」関係が強いことを意味します．

典型的な実験結果として，次のパターンが見られます：

  • \(h_{\max}\) と \(D_N^*\) は高い相関（\(\rho \approx 0.85\) 以上）を示す． Koksma–Hlawka 不等式の理論的な裏付けと整合します．
  • \(\bar{h} = 1/(N+1)\) は点数のみで決まるため， 同じ \(N\) で配置が異なる場合を区別できない． 「点数が同じでも誤差が大きく違う」ことを正しく予測できません．
  • \(\sigma_h\)（ギャップの標準偏差）は不均一さを測りますが， 絶対的なスケール（最大ギャップの大きさそのもの）を捉えきれません． \(h_{\max}\) より相関が弱くなることが多いです．
  • \(\rho = h_{\max} / \min_i \Delta_i\) は比率なので， 全体的にスケールが大きくても小さくても同じ値になります． FEM の形状正則性条件（最大最小比の上界）に対応しますが， 1次元では誤差のスケール自体を捉えるのに向きません．

log–log スケールと収束次数

セル 9 の両対数プロットで，Uniform と Chebyshev のデータ点が 参照線 \(\mathrm{err} \propto h_{\max}^2\) 付近に並ぶ場合， PINNs の誤差が \(h_{\max}\) の 2 乗オーダーで改善している可能性があります．

有限要素法の \(H^1\) 誤差が \(O(h)\)，\(L^2\) 誤差が \(O(h^2)\) であることと 比較してください． この対応が成立するなら，\(h_{\max}\) は PINNs における「有限要素メッシュ径」に 相当する量と見なせます．

Center-clustered の外れ値的な振る舞い

Center-clustered（特に幅が狭いもの）は， \(h_{\max}\) も \(D_N^*\) も非常に大きく，誤差も大きくなります． これらは外れ値的な挙動をするため， 相関の計算に大きく影響します． 全データの傾向と，均一系（Uniform, Chebyshev, Random）だけに限定した場合を 比較することも有益です．

理論的な位置づけ：何が残っているか

この連載の数値実験から浮かび上がるのは，次の仮説です：

仮説：十分に訓練された PINNs の近似誤差は， 最大ギャップ \(h_{\max}\) によって上から抑えられる． $$ \| u - u_\theta \|_{L^2(0,1)} \leq C(\theta_*) \cdot h_{\max}^{\alpha} $$ ここで \(\alpha > 0\) はある収束次数，\(C(\theta_*)\) は最適解 \(\theta_*\) に依存する定数．

しかし，この仮説には未解決の問いが多く残ります：

  • 「十分に訓練された」とは何を意味するか？ 損失がゼロになるほど訓練しても誤差が残る（spectral bias）ことが知られています．
  • \(\alpha\) は何か？ 問題の正則性（\(u\) の滑らかさ）に依存するはずです．
  • \(C(\theta_*)\) をどう評価するか？ FEM では \(|u|_{H^2}\) などのソボレフノルムが現れます．
  • 2次元・3次元では，残差点の配置の幾何はどう記述すべきか？ FEM におけるフラットネスパラメーターのような量が必要になるでしょうか．

これらが，この連載の先にある研究課題です．

この連載のまとめと今後

3回の数値実験を通じて，次のことを確認しました：

  • PINNs の PDE 損失は，残差の積分の数値求積近似である（第1回・第2回）．
  • 残差点の配置が変わると，同じ問題・同じネットワークでも誤差が大きく変化する（第2回）．
  • 最大ギャップ \(h_{\max}\) とスター乖離度 \(D_N^*\) が，誤差と最も強い相関を示す（第3回）．
  • 平均ギャップ（点数）だけでは配置の悪さを捉えられない（第3回）．

これは，有限要素法における幾何条件の研究と同じ問い方です： 計算から得られる幾何的量が，近似精度を保証するか？

1次元の最小例での探索は，ここで一区切りです． 次の段階は，2次元・高次元への拡張，および 厳密な誤差評価への接続です． それらは，ブログではなく論文として扱う予定です （Research Visions B1）．
